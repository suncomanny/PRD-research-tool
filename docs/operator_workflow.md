# PRD Research Operator Workflow

This is the current lowest-token operating model for running PRD Research.

The workflow is intentionally split so Codex handles deterministic local work and Claude handles only missing web collection.

## Code Words

Use these exact phrases with Codex / Claude:

- `PRD Research Prep`
- `PRD Research Collect`
- `PRD Research Finish`

## Canonical Paths

- Shared template:
  - `C:\Users\Sunco\Sunco Lighting\Product - Manny Tools\PRD Research\Templates\PRD_Research_Template.xlsx`
- Shared reports:
  - `C:\Users\Sunco\Sunco Lighting\Product - Manny Tools\PRD Research\Research Reports`
- Shared session root:
  - `C:\Users\Sunco\Sunco Lighting\Product - Manny Tools\PRD Research\output\research_sessions`

## Step 1. User

Fill the shared template.

Do not change the session folder manually.

## Step 2. Codex

Say:

```text
PRD Research Prep
```

Expected Codex behavior:

1. Read the canonical shared template.
2. Create a fresh session under the shared session root.
3. Seed Stackline and reference context.
4. Inspect session status.
5. Return:
   - the session path
   - a short pending-collection summary
   - the exact Claude collection prompt for that session

At this step, nothing should be published yet.

## Step 3. Claude

Say:

```text
PRD Research Collect
```

But use the full session-specific collection prompt generated by Codex, not only the code word itself.

Expected Claude behavior:

1. Preflight the exact mounted paths.
2. Read the existing session manifest/status.
3. Complete only the pending raw collection artifacts:
   - amazon
   - brick_and_mortar
   - brand_sites
4. Stop when collection is complete or clearly blocked.

Claude should not:

- patch repo code
- change templates
- rebuild reports
- publish outputs

## Step 4. Codex

Say:

```text
PRD Research Finish
```

Expected Codex behavior:

1. Use the existing session.
2. Run normalize, price enrichment, analysis, row reports, combined workbook, manifest update.
3. Publish the completed workbook into `Research Reports`.
4. Return the final published workbook path.

## Why This Split Exists

This is the current lowest-token model:

- Codex handles deterministic local work.
- Claude handles only external competitor collection.

This avoids burning Claude tokens on:

- repo exploration
- path discovery
- report rebuilding
- publishing

## Success Definition

A successful run means:

1. The shared template was processed.
2. Claude completed any missing raw competitor collection.
3. Codex finalized the session.
4. The completed workbook was published into `Research Reports`.
